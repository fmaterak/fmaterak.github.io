Selected Projects

ferrium

Rust

Distributed, strongly-consistent key-value store with the Raft consensus algorithm implemented from scratch.

  • Hand-rolled Raft consensus — leader election, log replication and safety guarantees, with no external consensus library.
  • Fully async on tokio with a Redis-compatible RESP2 protocol, so redis-cli and standard clients connect out of the box.
  • Automatic failover surviving minority node loss; pluggable LSM-tree storage, Prometheus metrics and TLS for client and inter-node traffic.
  • Deterministic simulation testing — network partitions and crashes reproduced in CI, not just described.

logpipe

Go

High-performance, zero-dependency CLI for real-time log streaming, filtering and pattern alerting.

  • Concurrent, stage-based pipeline built on goroutine pools and buffered channels, with natural backpressure instead of dropped events.
  • Regex and structured (JSON field) filtering plus sliding-window pattern alerting using O(1) ring buffers.
  • Benchmarked at 1.2M msg/s tail throughput with sub-millisecond p99 on a single core; ships as one static binary.
  • Interface-based plugin system for custom filters and outputs; text, JSON and CSV export.
  • Go
  • goroutines
  • CLI
  • Benchmarks
  • CI

Virtual Filesystem

C++23

Modular virtual filesystem in modern C++23 with an interactive shell and JSON persistence.

  • In-memory tree of files and directories driven by a Unix-like REPL (mkdir, cd, ls, touch, echo, cat).
  • Clean module boundaries — VFS, Node/File/Directory, Path and Serializer — with save/load to JSON via nlohmann/json.
  • Interface-based design and dependency injection; core modules covered by unit tests.
  • CMake project layout (src/, include/, tests/) targeting GCC 13+ / Clang 17+.
